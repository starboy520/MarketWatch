# 配置与安全

## 不要提交到 Git 的内容

- **`config.toml`**（本地复制自 `config.example.toml`）：常含飞书、LLM、X Bearer 等，已列入根目录 `.gitignore`。
- **抓包 / 调试大 JSON**、cookie、checkpoint 等：默认已忽略 `checkpoints.json` 等；勿手动 `git add` 敏感文件。

## 配置文件

1. `cp config.example.toml config.toml`
2. 按需填写各段；或通过环境变量覆盖（见下）。

可选：通过 `NEWS_AGENT_CONFIG` 指向其它路径的 TOML。

## 环境变量（密钥优先放环境）

| 变量 | 用途 |
|------|------|
| `DEEPSEEK_API_KEY` | DeepSeek API，覆盖 `[llm] api_key` |
| `FEISHU_APP_ID` / `FEISHU_APP_SECRET` / `FEISHU_RECEIVE_ID` / `FEISHU_RECEIVE_ID_TYPE` | 飞书应用发消息 |
| `NEWS_AGENT_X_BEARER` | X Web GraphQL 的 **public Bearer**；与 `config.toml` 里 `[x] public_bearer_token` **二选一必填**（代码中**不再内置**默认 Token） |

## X GraphQL Bearer

Guest 流程需要在请求头里带与网页端类似的 **Bearer**：

- 推荐：写入 **`config.toml` → `[x] public_bearer_token`**，或部署环境设置 **`NEWS_AGENT_X_BEARER`**。
- 须自行从合规渠道取得并轮换；**勿**将真实值写入仓库、日志或聊天截图。

若两者皆空，`create_x_client()` 会抛出 `ValueError` 并指向本文档。

## 轮换与泄露

若密钥曾在仓库、Issue 或聊天中暴露，请在对应平台**立即轮换**（飞书应用 Secret、DeepSeek Key、X Bearer 等）。
